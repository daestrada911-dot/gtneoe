About This Page & Course Integration

This page reflects both the technical skills we’re building and the requirements of the course. Here’s how it connects:

HTML so far
We’ve built semantic structures: headings (<h1>, <h2>), sections (<section>, <aside>), navigation buttons, and even a modal using <dialog>. These aren’t just for looks — they make our page accessible and organized.
CSS so far
CSS variables (:root) act like course “design tokens.” We’ve added gradients, animations, and responsive grids. The glow around the frame shows off how CSS can create visual emphasis with very little code.
JavaScript so far
We’ve added interactivity: theme toggles with localStorage, tab navigation with keyboard support, and a modal that explains CI/CD. A small “fun fact” is how clicking on my name changes its color — proving that even a single line of JS can personalize a site.
Course Requirements
  • Implement 1 HTML enhancement (tabs, modal structure).
  • Implement 1 CSS enhancement (design tokens, animated frame).
  • Implement 2 JS enhancements (theme persistence, accessible tabs).
  • Touch of JS: name changes to favorite color on click.
  • Validate HTML using W3C validator & test in Chrome/Firefox.
Fun Facts
  • <dialog> is a newer HTML element — not all courses teach it yet!
  • The glowing border uses a conic-gradient, which spins in CSS without images.
  • All animations run on the GPU, so they stay smooth even on slower laptops.
  • Every style can be changed by editing the :root variables in one place.

CI/CD (Quick Primer)

CI (Continuous Integration) automatically checks your code whenever you push (lint/validate/test). CD (Continuous Delivery/Deployment) automatically ships the latest good build to your site (e.g., GitHub Pages).

For this class: set up a GitHub Action to validate HTML and then deploy to Pages if it passes. That’s CI → CD.
